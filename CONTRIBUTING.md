# Contributing

Thanks for your interest in this tutorial repository.

This project was created while following training material by Paulo Dichone, as described in the README. Contributions should respect that context and avoid presenting tutorial-derived work as original product development.

## Before opening a pull request

- Raise an issue first for significant structural or scope changes
- Keep proposals focused on documentation, code clarity, compatibility, or clearly separated learning extensions
- Preserve attribution to the original trainer and course material where it already exists

## Good contribution areas

- Documentation corrections or clarity improvements
- Build or compatibility fixes needed to keep the tutorial project runnable
- Notes that explain concepts more clearly for future reference
- Small cleanup changes that do not obscure the tutorial intent

## Out of scope

- Reframing the repository as an original commercial project
- Removing or diluting attribution to the course or trainer
- Large rewrites that make the code harder to compare with the tutorial source

## Development notes

- Treat this repository primarily as a learning archive
- Keep changes readable and easy to review
- Prefer updates that preserve the educational value of the existing code

## Pull request expectations

- Use a clear title and short summary
- Explain the reason for the change
- Note whether the update is a compatibility fix, documentation improvement, or learning extension
- Keep pull requests focused and easy to review
